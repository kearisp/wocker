# Ініціалізація проєкту

Команда `ws init` використовується для ініціалізації нового проєкту в поточному каталозі. Вона створює конфігураційний файл `.wocker.json`, який містить основні налаштування проєкту та його інфраструктури.

```shell
ws init
```


## Створення проєкту

Спочатку потрібно перейти в директорію вашого проєкту:

```shell
$ cd ~/project-dir
```

У цій директорії виконайте команду `ws init`:

```shell
$ ws init
? Project name: (example)
```

Система запропонує вам ввести назву проєкту (за замовчуванням використовується назва поточної директорії).


### Тип проєкту

Далі необхідно обрати тип проєкту. Від цього залежить, як Wocker буде запускати ваш контейнер:

```shell
? Project type:  (Use arrow keys)
❯ Image
  Dockerfile
  Preset
```


#### Image

Оберіть цей варіант, якщо ви хочете використовувати готовий Docker image з Docker Hub або іншого реєстру.

```shell
? Project type:  Image
? Image Name:
```

Вам потрібно буде вказати повну назву образу (наприклад, `nginx:latest` або `mysql:8.0`).


#### Dockerfile

Оберіть цей варіант, якщо ваш проєкт має власний `Dockerfile`. Wocker самостійно побудує образ перед запуском.

```shell
? Project type:  Dockerfile
? Dockerfile:  (Use arrow keys)
❯ Dockerfile
  Dockerfile_dev
  dev.dockerfile
```

Система просканує поточну директорію та запропонує обрати один із знайдених Docker-файлів.


#### Preset

Пресети — це готові шаблони конфігурацій для популярних стеків технологій. Вони дозволяють швидко розгорнути проєкт із заздалегідь визначеними налаштуваннями.

```shell
? Project type:  Preset
? Choose preset: (Use arrow keys)
❯ apache
  go
  node
  php-apache
  shopify
```

Кожен пресет може мати власні додаткові запитання (наприклад, версія мови програмування або додаткові модулі).
